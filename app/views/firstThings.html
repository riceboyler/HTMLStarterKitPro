First Things First: ViewModels

Object Literal

                            define(function() {
                                var vm = {
                                    property: value
                                };

                                return vm;
                            });

  • Singleton Pattern: In Durandal, when you create a module using a singleton pattern, it is cached (aggressively), and there is only one available instance at any time in the app.

  • Be VERY careful about using when you use/don't use the Singleton Pattern. There are consequences...

Constructor Function

                        define(function() {
                            var vm = function() {
                                var self = this;

                                self.property = value;
                            };

                            return vm;
                        });

  • Constructor Function is typical Object Oriented pattern. Create as many instances of the module as you need.

  • Like the Singleton Pattern, for reasons of performance/memory usage, consider how many instances of a module you want running at once.

  • Make your life easier: use self and embrace the Revealing Module Pattern.
